# NPX Command

## Objective

Learn about the `npx` command and how to use it.

## Benefits

From the terminal, the `npx` command can directly run NodeJS modules that have been installed.

## Complete these tasks

1. In a VS Code terminal, navigate to this level folder.
2. Initialize a project with `npm`
3. Use `npm` to install `cowsay`
4. Use `npx` to run `cowsay` with `CodeX Academoooo!`.
5. View `package.json` and observe `cowsay`
6. Add `.gitignore` and ignore `node_modules`
7. Use `npx` to run `figlet-cli` with `Frontend Developer!`.
8. Confirm installation of `figlet-cli`.
9. View `package.json` and observe that `figlet-cli` is not there.
10. In `script.js`, use `console.log` to display a message.
11. Let the message explain `npx`.
12. Use `node` to run `script.js`.

## More Information

- The `npx` command can run an installed NodeJS module.
- The `npx` command temporarily installs NodeJS modules that aren't listed in `package.json`
- Packages should be installed and listed in `package.json` if they are important to the development of a project.

## Usage Tips

- Use the `npm install` command and the package name to install a package. Example: `npm install some-package`
- Use `npx cowsay` and a message to run the NodeJS module. Example: `npx cowsay "Hello World!"`
- Use `npx figlet-cli` and a message to run the NodeJS module. Example: `npx figlet-cli "Hello World!"`

## Hints

- `cowsay` works without quotes also.

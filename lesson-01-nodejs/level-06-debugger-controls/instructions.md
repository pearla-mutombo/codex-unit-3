# Debugger Controls

## Objective

Use the debugger to watch an error get caught.

## Benefits

You'll get a better understanding of how `try/catch` works.

## Complete these tasks

1. Write the code from scratch. Do not copy/paste.
2. In `script.js`, store a value with `localStorage`.
3. Use `document.querySelector` to select a `p` tag.
4. Wrap the code with `try/catch`.
5. In the `catch` block, use console.log to display a message.
6. Let the message explain how to use the debugger controls.
7. Run the script line by line with the debugger.
8. Inspect the `error` object in the `catch` block.

## More Information

- The `try` code block is skipped the moment an error occurs.
- The `catch` code block receives an `error` object.
- [try/catch reference](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/try...catch)

## Usage Tips

- The `catch` statement accepts an error object. Example: `catch(error)`
- [Debugger controls](https://code.visualstudio.com/docs/debugtest/debugging#_debug-actions)

## Hints

- Error objects sometimes have a `message` that can be displayed.
